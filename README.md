# Конфиг Neovim

Этот конфиг уже умеет:

- показывать ошибки и предупреждения в коде
- переходить к определениям и символам
- форматировать код
- открывать файловый менеджер, терминал и поиск по проекту

Главная клавиша в конфиге:

- `Leader` = `Space`

То есть запись `Space e` означает: нажать `Пробел`, потом `e`.

## Что нужно установить

Сам Neovim-конфиг не делает диагностику и форматирование в одиночку. Для разных языков ему нужны внешние программы.

Ниже таблица: что за что отвечает и что нужно поставить.

| Язык / файлы | Что работает | Что нужно установить | Команда установки |
| --- | --- | --- | --- |
| Python `.py` | ошибки, переходы, hover | `pyright` | `npm install -g pyright` |
| Python `.py` | type-check ошибок | `mypy` | `sudo apt install mypy` |
| Python `.py` | форматирование | `black` | `sudo apt install black` |
| JavaScript / TypeScript / JSX / TSX | ошибки, переходы, hover | `typescript-language-server` и `typescript` | `npm install -g typescript typescript-language-server` |
| Prisma `.prisma` | ошибки | `@prisma/language-server` | `npm install -g @prisma/language-server` |
| CSS / SCSS / LESS | ошибки | `vscode-css-language-server` | `npm install -g vscode-langservers-extracted` |
| JS / TS / JSON / YAML / HTML / CSS / Markdown | форматирование | `prettierd` | `npm install -g @fsouza/prettierd` |
| Lua | форматирование | `stylua` | установить `stylua` и добавить в `PATH` |
| Go | ошибки | `golangci-lint-langserver` и `golangci-lint` | `go install github.com/nametake/golangci-lint-langserver@latest` и `go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest` |
| Rust | ошибки, переходы, hover | `rust-analyzer` | установить `rust-analyzer` и добавить в `PATH` |

## Как устроен Python `.venv`

Если у тебя несколько проектов, у каждого может быть свой `dir/.venv`.

Конфиг старается сам найти ближайшее окружение для текущего Python-файла:

1. `.venv/bin/python`
2. `venv/bin/python`
3. `.venv/bin/mypy`
4. `venv/bin/mypy`

Это значит:

- можно работать сразу с несколькими проектами
- у каждого проекта может быть свой `.venv`
- Neovim должен брать ближайшее окружение относительно открытого файла

### Буферы и вкладки

Верхняя полоска с файлами в этом конфиге работает как список буферов через `bufferline`.

Полезно помнить:

- `Tab` и `Shift-Tab` переключают открытые буферы
- `Space x` закрывает один выбранный буфер
- `Space X` закрывает буферы справа
- `:bd` закрывает текущий буфер

Если открыт один файл и нужно просто закрыть именно его, самый прямой способ:

```vim
:bd
```

## Если что-то не работает

Сначала проверь:

1. установлен ли нужный бинарник из таблицы выше
2. доступен ли он в `PATH`
3. перезапущен ли `nvim` после установки

Полезные проверки:

```bash
command -v pyright-langserver
command -v mypy
command -v black
command -v typescript-language-server
command -v prettierd
command -v stylua
```

Если команда ничего не выводит, значит инструмент не установлен или не попал в `PATH`.
